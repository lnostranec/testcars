# Как открывается страница 404

Файл **404.html** уже лежит в корне проекта. Чтобы при ошибке 404 показывалась именно она, нужно настроить хостинг.

## Уже настроено в проекте

- **Apache** (общий хостинг с .htaccess): в корне есть файл **.htaccess** — при 404 будет открываться 404.html.
- **Netlify**: в корне есть **netlify.toml** — при 404 будет открываться 404.html.

## Без доп. настроек

- **GitHub Pages** — если в корне репозитория (или в папке `docs`) есть **404.html**, GitHub Pages сам показывает его при 404.
- **Vercel** — при деплое статического сайта файл **404.html** в корне автоматически используется как страница 404.

## Локально (на своём компьютере)

При открытии через `file://` (двойной клик по index.html) сервер не запускается, поэтому 404 не срабатывает.

Нужен локальный сервер, например:

- Расширение **Live Server** в VS Code / Cursor: «Open with Live Server» по index.html.
- Или в папке проекта в терминале:
  - `npx serve` (Node.js)
  - или `python -m http.server 8000` (Python)

После этого переход по несуществующему адресу (например, `http://localhost:3000/nesushhestvuyushhaya`) откроет 404.html.

## Nginx (свой сервер)

В конфиг сайта добавьте:

```nginx
error_page 404 /404.html;
location = /404.html {
    internal;
}
```

Убедитесь, что 404.html лежит в корне каталога, который отдаёт Nginx (например, в `root /var/www/html;`).
